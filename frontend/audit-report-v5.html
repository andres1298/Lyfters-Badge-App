Issues pendientes
CRÍTICO workspaces.py — event_joins huérfanos tras cascade delete
delete_workspace() elimina scans, badges, events, workspace_members e invitations, pero no elimina los registros de event_joins asociados a esos eventos. Consecuencias: (1) usuarios con joins huérfanos pasan el check de membresía en redeem.py para eventos inexistentes; (2) joined_events() puede retornar event IDs inválidos. Fix: agregar event_joins().delete_many({"event_id": {"$in": event_oids}}) en el pipeline de cascade.
MENOR redeem.html — botón Instagram sin fallback en desktop
El botón "Compartir en Instagram" usa navigator.share() directamente. En navegadores sin Web Share API (todos los desktop) el click no hace nada visible. scan.html sí tiene fallback de descarga PNG. Homogeneizar: agregar if (!navigator.share) { /* descarga o copia URL */ }.
MENOR events.py — /leaderboard sin limit() en MongoDB
El endpoint de badge leaderboard trae todos los usuarios de la colección, ordena en Python, y el cliente aplica .slice(0,8). Escala O(n) con n usuarios. Reemplazar con pipeline de agregación: $sort → $limit(8) directamente en la query.
MENOR api.js — Mock.getXpLeaderboard retorna XP=0 para todos
Mock.getXpLeaderboard accede a u.xp_total y u.level del schema de usuario mock, pero el mock guarda XP separado en mockState.xp. En modo mock el leaderboard de XP siempre muestra 0 XP / Nivel 1. Solo afecta la demo offline; producción usa el backend real.
